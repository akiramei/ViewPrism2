## 1. 症状

画像タブのチップ行(タグビュー軸の階層ナビゲーション)が**折返しもスクロールもせず、
ウィンドウ右端で切り捨て**られる。定義値展開(REQ-096)で 47 件級(都道府県)の値ノードを
展開すると、ウィンドウ幅に収まらない値ノードへ**いかなる操作でも到達できない**。

実測(2026-07-15・スクラッチ DB=`VIEWPRISM2_DATA_DIR` 差し替えで実ライブラリ不変):

| 条件 | 観測 |
|---|---|
| 最大化(2896px 級) | 47 チップ中 **20 件のみ可視**。山梨県以降は画面外 |
| 幅 1400px | **8 件のみ可視** |
| 通常ホイール(チップ行上) | 行は動かない |
| Shift+ホイール(水平) | 行は動かない |
| オーバーフローの視覚手がかり | なし(省略記号・スクロールバー・「他 N 件」表示のいずれも出ない) |

再現手順: textual タグ(候補値 47 件=都道府県)を作成 → ビュー階層ノードに配置し
展開モード=定義値 → 画像タブでビュー軸に切替え、該当ノードへ潜る。
再現 seed はスクリプト化済み(scratchpad/seed_img023.py 相当 — tags/textual_tag_settings/
views/view_tag_hierarchies の 4 INSERT。tag type は `textual`)。

**定義値展開の中核価値(付与 0 件でも全定義値を定義順で安定表示=ECO-086 裁定 d)が、
47 件級=REQ-095/096 の起票動機そのものの規模で沈黙的に機能しない。**

## 3. 切り分け済みの事実(確定/未検証の分離)

確定(実測済み):

1. mock のチップ行コンテナは `flex-wrap:wrap`(§2 表)= 折返しは設計済み・未設計ではない。
2. 実装の WrapPanel は横 StackPanel 内にあり折り返さない(47 チップで実測・§1 表)。
3. 水平/垂直ホイールとも行は動かない = 切り捨てられた分への到達手段が存在しない。
4. チップ行は中央ペインの固定クロム(image_tab.md「クロム固定・本体のみスクロール」)の一部。
   1 行時のレイアウトは折返し是正後も不変(WrapPanel の単一行配置は同一)= 既存 golden の
   視覚は不変見込み。

疑い(未検証・/eco-fix のプローブで確定させる):

- (a) 折返し有効化後、47 件級でチップ行が 3〜4 行に膨らみ、固定クロムがサムネ領域を
  圧迫する(mock flex-wrap の素直な帰結だが、行数上限は CAD 未定義= IMG-023 の範囲)。
- (b) `chipHint` ラベル(「階層を掘る」)と折返し 2 行目以降の左端整列の関係
  (mock はヒント含め同一 flex コンテナ= 2 行目はヒント幅を継がない)。

## 4. 是正方針(案・着手時確定)

真因構造そのものを消す(ECO-038 原則): チップ行から**横 StackPanel を撤去**し、
ItemsControl(WrapPanel)が**有限幅で測定される**コンテナ構成へ変える。
案= Grid 2 列(Auto=chipHint / *=ItemsControl)または chipHint を WrapPanel 内の先頭要素として
mock と同型化。折返し行数の上限や行内スクロール等の**新設計は持ち込まない**
(そこは IMG-023 = CAD 裁定待ち。本 ECO は CAD 既定義の折返しの回復のみ)。

プローブ(是正前赤の実測計画): headless で 47 チップのビューを描画し、
全チップの実描画矩形が可視幅内に折返し配置されること(是正前= 21 件目以降の矩形 X が
可視幅を超える/クリップされることを赤で固定)。GF-073/086 様式の余白・はみ出し実測系。
