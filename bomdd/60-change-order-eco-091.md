## 1. 要求(症状)

IMG-023A/B の裁定により、画像タブ(view 軸/FS 軸)と作業タブのチップ行の多量時設計が CAD で確定した
(正典= ViewPrismUI `17dc9f3`: 裁定資料 `docs/decisions/IMG-023-TAG-013-chip-overflow.md` §7・

確定契約(両タブ**同一契約・同時反映**=裁定 §7 受入条件 9):

1. 通常表示は**最大 2 行**(チップ行内スクロールなし・固定クロム高さ有界)。
2. 溢れは 2 行目末尾の**「ほか N 件」→ポップオーバー**(検索欄+全項目一覧・一覧の内部スクロール許容)。
   N=非表示項目数。**幅変更で表示件数と N を再計算**(選択・ナビゲーション状態は不変)。
3. view 軸=定義順・永続選択なし・ポップオーバー内含め**全ノード到達可能**・選択で閉じてナビ実行。
   FS 軸・作業タブ= **active+「クリア」は通常領域必須表示**(優先配置後は定義順)・
   ポップオーバー内選択は単一選択反映で閉じる。
4. キーボード契約: 「ほか N 件」はフォーカス対象・開くと検索欄→一覧の順・**Escape で閉じ
   「ほか N 件」へ復帰**・視覚順=フォーカス順。
5. 継続行は吊り下げインデント(B-a)= **現行 Grid(Auto,\*)の正式追認**(実装変更なしの見込み)。

## 4. 是正方針(案・着手時確定)

1. 両タブのチップ行に容量制御を実装(共有部品化は**しない** — LabeledChipStrip 抽出は裁定 §7
   「契約一致確定後」= 本 ECO の golden 後の別 ECO。本 ECO は同一契約の 2 面実装+共通受入)。
2. 実測折畳み: 全チップ配置後の実描画矩形から 2 行に収まる可視件数と N を算出
   (mock アルゴリズム移植・ECO-027 LayoutUpdated 様式)。幅変更で再計算・選択/ナビ状態不変。
3. 「ほか N 件」ボタン(2 行目末尾・フォーカス対象)+ポップオーバー(検索欄+全項目一覧+
   内部スクロール)。view 軸=選択でナビ・FS/作業=単一選択反映。Escape でボタンへフォーカス復帰。
4. i18n: `chip.moreItems`(ほか {n} 件)等を 3 層(XAML/VM 算出/検査)で追加(ECO-080 規約)。
5. 受入: CP-CHIPWRAP-088 を容量・到達性契約へ拡張(裁定 §7 受入条件 1〜6・9 を検査次元化)。
   probe は両タブへ同一ベクトル(ECO-090 の共通パラメーター化と合流可 — 実施順は fix 時判断)。

プローブ(是正前赤の実測計画): 47 件級で「チップ領域の行数 ≤2」(是正前=8 行級で赤)・
「N=非表示数一致」・「active/クリアの通常領域残留」(FS/作業)・「ポップオーバー経由の全ノード到達」。
