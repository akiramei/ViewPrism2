# Change Order — ECO-050(staged): 類似しきい値の既定値が仕様と不一致(整理トレイ 80/作業タブ 90 — 仕様既定 70)

> ECO-048 golden 実施時の R3 所見(51-cheat-log 2026-07-06)から昇格起票。maintainer 実機で
> 「規定値は 80」と認識された時点で顕在化(仕様の既定 70 がユーザー認知からも失われていた)。

## 1. 症状(maintainer 実機・2026-07-06)

- 画像タブ整理トレイの類似検索しきい値の既定が **80**(ECO-048 golden で実測)。
  仕様(REQ-064/065・§2.10.4)は**既定 70**・範囲 50〜100。
- 実害: 既定 80 のままでは、回転複製(スコア 70 台 — ECO-048)や異フォーマット複製
  (jpg×png=78 点 — 51 R3 #2 の経路系統誤差)が**既定で検出から漏れる**。70 へ下げれば拾える
  (maintainer 実機で確認済み)。

  grep 実測: 「閾値スライダー」の存在のみ)= **工場の仮置きが仕様と突合されないまま混入**。

## 3. 切り分け済みの事実(確定と未検証を分離)

確定(コード読解・履歴):

- 実装の既定値は三者三様:
  - 独立モーダル SimilarSearchViewModel.cs:98 `_threshold = 70` = 仕様どおり
  - 整理トレイ ImageTabOrganizeViewModel.cs:33 `_similarThreshold = 80` = 逸脱
  - 作業タブ WorkTabViewModel.cs:75 `_similarThreshold = 90` = 逸脱(整理トレイ 80 とすら不一致)
- 混入コミット(`git log -S` で確定):
  - 80 = `0cf44ba`「整理モード製造①(M3-backbone)」(ECO-014 系・整理トレイ導入時)→
    `729039c`(ECO-036 切り出し)で ImageTabOrganizeViewModel へ移設・値は温存
  - 90 = `81500a8`「作業タブ(ECO-020/021 β)」= 整理トレイからの転写時に**さらにドリフト**
- マスキング(潜伏要因): 既定値は CP unit(エンジンは明示閾値で検査)にも golden チェックリスト
  (トレイの見た目・操作は G-9/G-1 で承認済みだが既定値は明示観点でない)にも載っておらず、
  機械・実機のどちらの検査面にもなかった。潜伏 = `0cf44ba` から本日まで。
- **追加所見(R3 候補)**: 仕様適合の 70 を持つ唯一の実装=独立モーダル
  (SimilarSearchViewModel/SimilarSearchWindow)は **WindowService.ShowSimilarSearchAsync の
  呼び出し元がゼロ=到達不能コード**(ECO-014 のモーダル→トレイ置換の残骸疑い)。
  本 ECO には混ぜない(51-cheat-log へ記録・撤去は ECO-024 型の別件)。

未検証(疑い):

- 作業タブの 90 に意図(スペース内は高精度で絞る等)があった可能性 — order/裁定記録に痕跡なし
  (bare literal・コメントなし)のため「意図なきドリフト」と判定しているが、maintainer の記憶に
  裁定があれば gate で申告を(その場合は spec 改訂=裁定案件に転じる)。

## 4. 是正方針(案 — 実装逸脱につき裁定不要・最小是正で着手可)

- **最小是正(推奨)**: 両 VM の既定値を 70 へ(literal 2 箇所)+ 既定値を pin する受入テスト
  (両 VM の初期 `SimilarThreshold == 70` = 仕様既定の unit 固定 — 今回の潜伏様式「既定値が
  どの検査面にもない」を直接塞ぐ)。範囲 50〜100・整数刻みは既存実装のまま。
- 根拠: 仕様が正(R2 上流優先)。かつ ECO-048 の実測が 70 の実用性を補強
  (回転複製 70 台・異フォーマット複製 78 点は既定 80 で漏れる)。
- 代替(採らない): 用途別既定(整理 80/作業 90)の正式化 — spec 改訂を要する裁定案件になるが、
  値に意図の痕跡がなく、漏れの実害が実測済みのため提案しない。
- プローブ(R5): 既定値 pin テストを先に追加し、**是正前に 80/90 で不合格**を実測してから 70 へ。
