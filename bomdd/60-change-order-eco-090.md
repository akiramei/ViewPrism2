# Change Order — ECO-090(チップ行同期実装の暫定統制 — E-BOM 関係宣言+CP-CHIPWRAP-088 の両タブ共通パラメーター化)

- 起票日: 2026-07-15
- 報告者: メンバーレビュー助言(2026-07-15・ECO-088/089 クローズ後の依存関係整理)→ AI が台帳実測で谷間を確定
- 種別: 予防処置(検査器/台帳の欠陥 — 製品コードの欠陥ではない。src 無変更・tests+bomdd のみ)
- status: staged

## 1. 症状(要求)

ImageTabView と WorkTabView のチップ行(ヒント見出し+WrapPanel)は**同一意味論の同期実装**
(別ファイルに同型コードが 2 面)だが、その関係がどの台帳にも宣言されていない。
ECO-088 で片側(ImageTabView)だけ是正し WorkTabView を見落とした read-across 漏れが
実害として実証済み(ECO-089 findings・教訓 1)。

共有部品化(DRY 統合)は IMG-023A/B・TAG-013 の CAD 裁定
(ViewPrismUI `docs/decisions/IMG-023-TAG-013-chip-overflow.md`・VPUI `85943b6` で起票済み)が
確定するまで保留(ECO-089 §6 残置)のため、**統合までの間、次の修正で再び片側だけ直す構造が
残っている**。コードを共通化せず受入検査と台帳宣言を共通化する中間措置が必要。

## 3. 切り分け済みの事実(確定/未検証の分離)

確定(実測済み):

1. E-UI-AXIS-NAV-040 / E-UI-WORKSPACE-043 の invariants・graph_edges に相互参照なし
   (双方向の同期実装宣言が不在)。ECO-089 追補で説明文に「read-across 閉包」の事後記録は
   あるが、**将来の変更 ECO に read-across を義務づける前向きの宣言ではない**。
2. CpUi088ChipWrapTests(画像タブ=47 件/5 件)と CpUi089ChipWrapTests(パレット 8/2 件・
   作業タブ 47/3 件)は**別ファイル・別 seed・別アサーション実装**で、同一データを両タブへ
   流す構造になっていない。
3. 両タブのチップ行は XAML 構造が同一(Grid Auto,* + chipHint + ItemsControl/WrapPanel
   LineSpacing=8)・ChipVM も共用 — 検査の共通パラメーター化は技術的に成立する。
4. タグパレット候補値行は**同一様式(WrapPanel 測定文脈)だが同一意味論ではない**
   (非対話プレビュー・TAG-013 で独立裁定待ち)。同期実装ペアは両タブの 2 面。

疑い(未検証・fix のプローブで確定させる):

- (a) 新次元(長ラベル・狭幅・境界件数)の probe が現行実装で緑か赤かは未実測。
  **赤が出た場合は新欠陥の発見であり、本 ECO で直さず R3 分離起票する**
  (本 ECO は検査の谷間を埋める工事。是正は別 ECO)。
- (b) 狭幅ウィンドウの具体値は ECO-027(ツールバー段階収納)の発火幅と干渉しない値を
  fix 時に実測で確定する。折返し境界件数(1366px で 1 行に収まる最大件数)も実測で確定。

## 4. 是正方針(案・着手時確定)

1. **E-BOM 関係宣言**: E-UI-AXIS-NAV-040 と E-UI-WORKSPACE-043 の双方の invariants へ
   「チップ行(ヒント見出し+WrapPanel)は両 surface の**同一意味論の同期実装** — 一方の
   チップ行を変更する ECO は、他方への read-across 結果(適用 or 非該当の理由)を ECO 本文に
   必須記録する。共有部品化は IMG-023A/B 裁定後の DRY 判断(ECO-089 残置)まで保留」を追記。
2. **受入検査の共通パラメーター化**: 両タブへ**同一のテストベクトル**を流す共通 probe
   (Theory 化 or 共通ヘルパ+ビュー種別パラメーター)へ再構成:
   1 件 / 5 件(mock デモ規模=1 行 pin)/ 折返し境界件数(境界の両側)/ 47 件(起票動機の規模)/
   長いラベル / 狭幅ウィンドウ。既存 CpUi088/089 のアサーション(right≤可視幅・行数・
   アイコン可視)は資産として維持し、seed とウィンドウ構築を共通化する。
   タグパレット面の probe(CpUi089 面 A)は現状維持=対象外(TAG-013 裁定後の反映 ECO に譲る)。
3. **CP 更新**: CP-CHIPWRAP-088 の test_vectors を共通ベクトル表記
   (面×件数×ラベル長×幅のマトリクス)へ更新し、characteristic へ本 ECO の統制目的
   (統合前の同期実装ガード)を追記。
