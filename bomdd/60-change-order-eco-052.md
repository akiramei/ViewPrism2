## 2. 工程診断 — CAD 未定義+台帳内不整合(三層)

| 工程 | 判定 | 根拠 |

| E-BOM | **項目内で矛盾** | E-UI-MERGE-036: 冒頭 invariant はプレビュー+非破壊明示を要求・ECO-014 invariant のトレイ構成(マージ先/整理対象/実行/完了)にはどちらも無い |

- **マスキング**: ①旧 MergeDialog に実装が残存し「REQ-067 実装あり」に見えた(grep すると死者がヒット)
  ②G-9 チェックリストが旧モーダルの操作手順のまま as-built 未同期(「グリッドで画像を 1 枚選び

- 結論: **どちらが正か(要求を充足するか・as-built を追認するか)の裁定が必要**。CAD 未定義のため
  a 案は CAD 是正(ViewPrismUI)が先(R2 上流優先)。

## 3. 切り分け済みの事実(確定と未検証を分離)

確定:

- プレビューの目的(誤マージ防止)に対する現行トレイの代替装備:
  ①検索結果は候補+一致率の**見比べ重視**表示(ECO-014 設計意図)
  ②トレイにマージ先/整理対象のサムネイル一覧(実行前に対象は視認可能 — タグの統合結果だけが不可視)
  ③**「取り消す」= ECO-044 補償 Undo**(事後の完全復旧手段・golden approved)
  → 「実行前タグプレビュー」の保護価値は V3 モーダル時代(Undo なし)より相対的に低下している。
- 非破壊明示の近縁前例: ゴミ箱の完全削除は『画像ファイルは削除されません(DB から除去)』の明示+確認
  あり(G-10・CAD work_tab.md L125 で文言まで裁定済み)。マージにだけ明示が無い。
- G-9 チェックリスト自体の as-built 未同期(モーダル手順のまま)は、本件の裁定と独立に doc 是正が必要
  (どの案でも実施)。

未検証:

- 原典(view-prism)の ImageMergeModal がプレビュー+非破壊注記を持っていたか(REQ-067 の出典 —
  rationale は原典 UI 由来。トレイ化での脱落が「原典等価性の低下」でもあるかは原典リバースで確認可能。
  裁定の参考情報であり必須ではない)。
