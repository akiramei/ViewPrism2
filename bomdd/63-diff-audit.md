# Diff Audit — <ECO/CAPA-ID> 不要改変監査(Phase 7)

> **[disposition: 未採用テンプレート / unused-template]**
> 本ファイルは BomDD テンプレ一式の不要改変監査テンプレ。**本プロジェクトでは独立 working artifact として使っていない。**
> 不要改変監査は各 ECO 本文(`60-change-order*.md` の §5 回帰+変更受入)に直書き運用している(例: [ECO-002 §5](60-change-order-eco-002.md) は Run5 で 63 相当を実施し unnecessary modification 0 を記録)。
> ECO 索引は [60-change-register.yaml](60-change-register.yaml)、正本宣言は [00-manifest.yaml](00-manifest.yaml) を参照。
> 下記テンプレ本体は参照用に保持する(隔離工場へ diff 監査を事前宣言する標準 ECO 用の雛形)。

> 部分改修の**変更境界はオラクルに映らない**(影響なし箇所を壊さずに書き換えることは可能=回帰緑のまま境界違反が通る)。**diff で測る**。
> 規律: 「影響分析にある箇所だけを改修せよ。影響なし箇所への変更は禁止。diff を測定する」を work order に**事前宣言**する(「測定されると知っている」ことの規律効果を含む。forward-01.5 では 2 工場とも不要改変 0)。

## 1. diff 基準点
- 改修入力(変更前ソースの複製)を工場別ディレクトリへ**設計者が事前コミット** = 基準点(改修が複製に混ざらない)
- 納品後: `git diff <基準点>..<納品>` を影響分析([61-impact-analysis.md](61-impact-analysis.md))と突合し、影響分析**外**への変更を抽出する

## 2. 不要改変の分類(影響分析外への diff)
| 分類 | 内容 | 扱い |
|---|---|---|
| format/noise | 整形・コメント・空白のみ | 記録のみ |
| test-only | 受入ハーネス・テストコードのみ | 記録のみ |
| behavior-risk | 挙動に影響しうる変更 | 要審査(回帰被覆の内か外か) |
| contract-change | 公開契約の変更 | **違反**(ECO 外の契約変更) |
