# ECO-002 出所分析 — golden 指摘(G1〜G7)はどの工程で混入したか

日付: 2026-06-12 / 分析者: 設計AI(承認者 maintainer の問題提起による)

## 問い
G1〜G7 の各指摘について、原因を次の仮説で帰属する:
- (a) 初期仕様+porting-spec に**情報自体が無かった**
- (b) 情報はあったが**曖昧な表現で解釈にぶれ**が生じた
- (c) 情報は存在したが**当時読み取れなかった**(抽出漏れ)
- (d) **読まずに推論**した(トレース偽装)
- (e) 上記以外(製造欠陥・知識欠落・仕様の沈黙・検査深さ等)

## 仮説への回答

- **(a) 情報自体が無かった**: ほぼ該当なし。唯一 SHIFT 範囲選択が porting-spec に無いが、**原典ソースには有る**(リバース工程で設計者は原典を読めるので入手可能だった)
- **(b) 曖昧表現→解釈ぶれ**: G1③(「日付」)、G1①(prop 既定値→ユーザー UI への昇格は解釈拡大の一種)
- **(c) 存在したが読み取れなかった**: G1②(抽出漏れ)、G1④(スコープ次元の脱落 — 最も重い)
- **(d) 読まずに推論(サボり)**: **検出されず**。G1① の行番号引用は正確で、§2.6 の「原典はフォルダ風表示+パンくず」という記述は原典ナビを読んでいた証拠。トレースの誠実性は保たれていた
- **(e) 分類外**: G3(チェックリスト)、G4(製造+検査深さ)、G5(知識パック)、G6(沈黙+製造) — **製造系 4 件は情報パイプラインの問題ではない**

## メタ所見

1. **要求系(G1)と製造系(G4-G6)で帰属がきれいに分かれた**。要求系はすべて Phase 2 リバース仕様化の品質(新旧 UI 区別・次元の全数照合)に、製造系は知識パック・沈黙次元・検査深さに帰着する。
2. **golden は設計どおり機能した**。特に G1②(どの文書にも無く golden でしか出ない)と G1④(2 つの監査を素通りしたものを捕捉)は golden の存在価値の実証。
3. **ECO-001 突き合わせの教訓(設計AI自身)**: 「コアは原典準拠」という結論は正しかったが、照合をコア偏重で打ち切った。RVP-\* 行(横断不変条件 4+領域別 30 余)の**全数照合をチェックリスト化**して再実施すべき。G1④型の脱落が他にもある可能性が高い — 既に候補を 3 件発見:
   - 02§1「**選択中のコレクションは永続化し、次回起動時に復元**」(V1 未実装 — CR-2 の仕様化に含めるべき)
   - 02§2「**表示モード(グリッド/リスト)はユーザー設定として永続化**」(V1 settings に無い)
   - 02§2 ビューア「ESC **または背景クリック**で閉じる」(V1 は Esc+閉じるボタンのみ)
