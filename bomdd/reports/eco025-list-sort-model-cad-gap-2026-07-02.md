# CAD フィードバック: ファイル一覧「リスト表示のソートモデル」記述漏れ（ECO-025 β golden 由来）

- **日付**: 2026-07-02
- **宛先**: ViewPrismUI 担当（CAD 権威）
- **発信元**: ViewPrism2 実装（ECO-025 β・画像タブ ファイル一覧の列描画＋列ヘッダーソート）
- **区分**: UI-IR / プロース CAD の記述漏れ（実装ミスでも BOM ミスでもない）。**モックは正**。
- **失敗型**: S3（データ分散レイアウト/インタラクション欠陥＝モックには在るが UI-IR が契約として抽出していない）。VP-UI-007（layoutInvariant 横展開）と同族。
- **重要**: 本件は CAD（`docs/screens/*.md`）側の是正依頼です。ViewPrism2 側は CAD 確定後に BOM→実装へ反映します。ViewPrism2 が先行して修正すべき項目ではありません。

## 1. 観測された症状（ViewPrism2 実機）

maintainer 実機レビュー: 「タグタブ『ビューを編集』で**表示列に追加したタグ列（例: 評価）が、ファイル一覧でソート対象として選べない**」。

## 2. 事実確認: 機能は正常＝「導線（入口）」の問題

ViewPrism2 では列ヘッダーソートは**機能的には正常**（統合テストで `ヘッダークリック → 型別比較器 → 行並べ替え` を確認済）。したがって症状は**機能欠陥ではなく、ソートの入口が分かりにくい導線問題**。

原因の切り分け:
- ViewPrism2 実装はリスト表示でも**ツールバーの「名前 / 更新日 / サイズ」ソートメニューを残していた**。このメニューには**タグ列が無い**。
- 列ヘッダー自体は押せる（＝ソート対象）だが、**クリック可能なアフォーダンスが弱く**、ヘッダーがソートボタンだと気づけない。
- → ユーザーはツールバーのソート（タグ列なし）を見て「タグ列は選べない」と判断した。

## 3. モックは正しい（ファイル一覧 standalone）

一次モックのリスト表示ツールバーは:

```
表示[サンプルアルバム]  |  表示列 4/5  |  （右）↓ 評価（降順） ✕ チップ  |  グリッド / リスト
```

- **名前 / 更新日 / サイズ のソートメニューは無い**。
- ソートは**列ヘッダー**で行う（アクティブ列にラベル強調＋方向矢印）。
- アクティブなソートは**ツールバーのチップ「↓ <列名>（方向） ✕」**で要約し、✕ で解除。

→ モックでは「表示列（タグ含む）＝ソート対象」であることが**構造的に自明**。

| モック | **正** | 上記（ヘッダーソート＋要約チップ・名前/更新日/サイズ メニュー無し） |

## 5. CAD の具体的な記述漏れ・不整合

1. **list モードのツールバー・ソートモデルが未抽出**
   - `docs/screens/image_tab.md`（親・ツールバー所有）は **`ソート | クリア、名前、更新日、サイズ`**（154 行付近）、**`ソートメニュー | 名前、更新日、サイズでソート`**（298 行付近）を**無条件に**記載＝実質グリッドのモデル。
   - `docs/screens/file_list.md`（リスト詳細）はソート＝**列ヘッダー**＋「ソート概要（クリア）」（95〜103・137〜138 行）だが、レイアウトのツールバー列挙（49 行「表示軸・表示列ボタン・グリッド/リスト等」）に**ソート控えを持たない**。
   - **どちらの CAD も「リスト表示ではツールバーの名前/更新日/サイズ ソートメニューを出さず、ソートは列ヘッダー、アクティブソートはツールバーのチップで要約する」という list モードの切替を明記していない**。
2. **image_tab.md ↔ file_list.md の不整合**
   - image_tab（ツールバーにソートメニューあり）と file_list（ヘッダー＋ソート概要）が食い違う。グリッド＝ツールバーメニュー / リスト＝ヘッダー＋チップ、という**モード別の対応関係が明文化されていない**。
3. **列ヘッダーのソートアフォーダンスが未規定**
   - 「各列ヘッダーはソートボタン」とはあるが、**押せると分かる見た目**（Hand カーソル／ホバー／非アクティブ列の弱い並べ替え示唆）が契約になっていない。

## 6. 推奨 CAD 追記（依頼）

`docs/screens/file_list.md`（および必要に応じ `image_tab.md`）に、**リスト表示のソートモデル**をインタラクション契約として明記:

- **(a)** リスト表示では、ツールバーの「名前 / 更新日 / サイズ」ソートメニューを**出さない**（それはグリッド表示のモデル）。リストのソートは**列ヘッダー**が担う。
- **(b)** ソートは各**列ヘッダー**をクリック（新規列＝昇順開始 / 同列再クリック＝昇順⇄降順）。**ヘッダーはクリック可能と分かるアフォーダンス**（カーソル／ホバー／非アクティブ列の弱い並べ替え示唆・アクティブ列の方向矢印）を持つ。
- **(c)** アクティブなソートは**ツールバーのチップ「↓ <列名>（昇順/降順） ✕」**で要約し、✕（クリア）で解除。「ソート概要」＝このチップである旨を明記。
- **(d)** image_tab.md と file_list.md の**モード別ソートモデルの対応**（グリッド＝ツールバーメニュー／リスト＝ヘッダー＋チップ）を両 CAD で整合させる。

これらは VP-UI-007 と同じ「モックにはあるが UI-IR 未抽出」の是正であり、可能なら**インタラクション契約 or layoutInvariant の隣接節**として固定し、テンプレの必須チェックにも「ソート/操作の入口（どのコントロールが操作対象か）」を加えると再発防止になります。
