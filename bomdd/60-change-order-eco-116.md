# ECO-116: タグ編集パネル「タグ追加」タブの末尾が表示しきれない — スクロールが最後まで到達しない

- 起票日: 2026-07-20
- 報告者: maintainer(実機観測・スクリーンショット添付)
- baseline: main `c258030`
- 種別: 不具合是正候補(レイアウト/実装層見込み。真因は未確定=プローブ先行で確定させる)

## §1 症状

タグ編集モードの右ペイン(タグ編集パネル)「タグ追加」タブで、
**スクロールバーは表示されるが最後まで移動せず、末尾のコンテンツが表示しきれない**
(2026-07-20・maintainer 実機)。

- 観測時の状態: 1 枚選択中・「タグ追加」タブ。
- スクリーンショットの見え方: 上端でシンプルタグ行(「ばにしゅ!」)が途中で切れている
  =スクロール済み。下端は「数値 値を選ぶ」グループ見出しの直後で切断され、
  数値タグ行が見えない。
- 報告者の言い回し: 「見た目はスクロールバーが最後まで移動しないように見えます」
  =つまみが下端に到達しきらない/到達しても末尾に届かない。

## §2 工程診断(R2)

## §3 切り分け済みの事実

### 3.1 確定(起票時・コード実測)

**対象マークアップ**: [ImageTabView.axaml:144-309](../src/ViewPrism2.App/Views/ImageTabView.axaml)
(タグ編集パネル「アクティブ(選択あり)」の `DockPanel`)。子は 4 つ:

| # | 要素 | Dock | 備考 |
| --- | --- | --- | --- |
| 0 | タブ `Grid`(:146) | Top | 現在のタグ/タグ追加 |
| 1 | 区切り `Border`(:152) | Top | 高さ 1 |
| 2 | **現在のタグ `ScrollViewer`(:155)** | **無指定** | `IsVisible={Binding OnCurrentTab}` |
| 3 | **タグ追加 `ScrollViewer`(:192)** | **無指定** | `IsVisible={Binding OnAddTab}`・`Padding="16,14,16,20"` |

- 症状面(タグ追加)は **DockPanel の最終子**= `LastChildFill` の対象。
- **現在のタグ(#2)は Dock 無指定の非終端子**= **`Dock.Left` 既定**。
  これは **ECO-037 で「幅算定が壊れる」と確定済みの構造パターン**であり、
  ECO-037 の是正(`Panel` で包んで最終子化)は**整理トレイ側だけに適用され、
  タグ編集パネルには read-across されていない**
  (是正の痕跡= [ImageTabView.axaml:329-333](../src/ViewPrism2.App/Views/ImageTabView.axaml) のコメントと `Panel` 包み。
  タグ編集パネル側には同等の包みが無い)。
- **面間対称**: [WorkTabView.axaml:141-…](../src/ViewPrism2.App/Views/WorkTabView.axaml) が**同一構造**
  (同じ 4 子・同じ Dock 無指定 2 枚)。**非対称ではなく同型**=是正時は両面同時(§8.2 閉包)。
- **容器側は健全**(高さ無制限の疑いを潰した実測):
  `MainWindow.axaml:222` の `Panel` は `DockPanel` の最終子で、上=タブバー・下=ステータスバー(高さ 28)
  に挟まれ**高さは有界**。`ImageTabView.axaml:14-15` の root も `Panel`+`DockPanel`。
  → **「右ペインがウィンドウより背が高く、下端がクリップされている」系の疑いは容器レベルでは否定**。
- **下部 docked アクションは非該当**(高さを食う要素の見落としではない):
  CAD 行 107 は「タグ編集の付与操作…は下部 docked」と書くが、
  同 CAD 行 737/789-791 が **「行クリックで即時付与」** を正典としており、
  タグ編集パネルに docked ボタンは存在しない(実装も無し)。
  → 本 ECO のスコープ外。ただし **CAD 内部の記述不整合**として §6 に分離記録。

### 3.2 未検証(疑い・fix 時に実測で確定)

真因候補は 2 系統。**どちらかを実測で先に落とす**のが第 1 手。

- **候補 A(退行・時間的相関が強い)**: **ECO-115(`a2eb10e`・2026-07-20 本日)の副作用**。
  同 fix は本症状と**同一パネル・同一タブ経路**(`TabAdd`/`ClickAddRow`)から
  `Recompute()`(=`Items` 全再構築+大量 `CollectionChanged`)を撤去し、
  `BuildContextPanels`+`OnPropertyChanged(string.Empty)` に置換した。
  疑い= **従来は全面再構築の巻き添えで起きていた測り直しが失われ、
  ScrollViewer の Extent が旧内容のまま滞留**(=つまみが短い範囲しか動かない)。
  ECO-115 の probe はインスタンス同一性(構造)であり、**レイアウト量(Extent/Viewport)は未検査**
  =検査の谷間として整合する。
- **候補 B(潜伏・構造)**: §3.1 の **非終端子 `Dock.Left`** 構造に起因する測り直しの取りこぼし
  (ECO-037 と同根で、あちらは幅・こちらは高さ/Extent に出る変種)。
  ECO-115 以前から潜伏していた可能性。

**この 2 つは 1 回の A/B で分離できる**(§4-1)。

## §4 是正方針(着手時確定・プローブ先行)

1. **切り分け A/B(最初にやること)**: `7d5e0fd`(= ECO-115 直前)個体で同一操作を再現するか。
   - 再現**しない** → 候補 A 確定(ECO-115 の退行)。是正は通知/測り直しの補完。
   - 再現**する** → 候補 A 却下・候補 B へ(構造是正= `Panel` 包みの read-across)。
2. **プローブ(赤→緑転)**: headless で `ScrollViewer` の
   **Extent.Height / Viewport.Height / Offset.Y** を実測し、
   「**Offset を最大まで送ったとき、最終要素の下端がビューポート内に入る**」を検査する。
   固定ピクセル閾値ではなく**到達性の関係式**で書く(ECO-058 様式)。
   タブ切替・行展開(候補値/数値の `inlinePicker` 展開でコンテンツ高が増える経路)も対象。
3. **是正**: 1 の結果に従う。**ImageTab / WorkTab 両面同時**(§3.1 の同型・§8.2 閉包)。
4. **R7(セルフゴールデン)**: パネル末尾の可視性は視覚契約 → captures 並置の対象。
