## 題材
- 何を作るか(1〜3行):
  TypeScript/Electron 製画像管理アプリ view-prism(C:\Demo\source\repos\view-prism)を
  .NET 10 / Avalonia UI 12.0 / SQLite で再実装する。
  中核コンセプトは「タグ × 仮想ビュー」によるフォルダ階層に依存しない画像管理。
- 種別: GUI(デスクトップアプリケーション)
- 黒箱境界:
  - 核(ドメインロジック・DB層): in-process(観測契約が必要)
  - 表面(画面描画・操作): GUI(golden+承認者が必要)

## スコープ
- 含む — 第1段ループ(仮、Phase 1 終了時 G1 で確定):
  - 同期フォルダ管理・画像の増分スキャン・ハッシュ計算
  - タグシステム(simple/textual/numeric、階層、色)
  - 仮想ビュー(条件フィルタ、タグ階層ナビゲーション)
  - サムネイル生成・グリッド表示・リスト表示
  - 基本画像ビューア(単一画像フルスクリーン)
  - SQLite 永続化・マイグレーション基盤
  - i18n 基盤(ja/en)
- 含まない — 後続ループ / ECO で追加(理由付き):
  - 類似画像検索・重複検出(OpenCV ネイティブ依存が重く、リスク最大の部品のため分離)
  - 見開き・スクロール・ストーリービュー・タグ制御モード(表面難度が高く、ビューア拡張ループとして分離)
  - ワークスペース・Undo/Redo(コア確立後の機能追加として分離)
  - バックアップ/復元・DB修復・ファイル再リンク(運用系機能のため後続)
  - 既存 viewprism.db からのデータ移行(新規DBで開始。必要になれば ECO + 62-migration-oracle で対応)
  - macOS/Linux 対応(対象は Windows のみ。Avalonia 採用により将来の道は残る)
