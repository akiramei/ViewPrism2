## 1. 症状(2026-07-06・Phase 0 三者差分)

- 整理トレイ「似た画像を探す」の条件検索に入力欄が **2 つしかない**(ファイル名・拡張子)。
- CAD(v1 モックから一貫)は **5 条件**: ハッシュ値/拡張子/サイズ/ファイル名/更新日
  (v1・v2 モックの `condDefs` は完全同一 — v2 で増えたのではない)。
- 実害: 完全重複の検出(ハッシュ値一致)・日付や サイズでの絞り込みが整理トレイから使えない。

## 7. 実施記録(2026-07-06 — 是正・機械受入完了・golden 待ち)

- **診断の拡張(fix 中の発見・R5 停止→裁定)**: モック(v1/v2 同一)の照合ロジック実測で、条件検索の
  意味論は自由入力ではなく**「マージ先(dest)との属性一致をトグルで選ぶ」**と確定
  (`s.cond.ext && im.ext !== dest.ext` 等)。つまり本 ECO は「3 条件の欠落」ではなく
  **「意味論の取り違え(自由入力 2 欄)+条件 3 種の欠落」**だった。モックの照合式はデモ用ダミー
  (減点式)のため、各属性の「一致」の定義は沈黙次元 → 実装前に停止し裁定を得た。
  なお撤去した旧 BuildCriteria には「hash/サイズ/更新日は surface 増分(M2)で結線する」の
  コメントが残存 — **宣言された増分が実施されないまま潜伏**していた(マスキング要因の追加)。
- **裁定(maintainer 2026-07-06)**: ①一致の定義= ハッシュ SHA-256 完全一致/拡張子 完全一致(大小無視)/
  ファイル名= dest 名本体(拡張子除く)の部分一致/サイズ= バイト完全一致/更新日= 同一日(UTC)
  ②自由入力 2 欄は撤去し全面置換(a)③条件検索もマージ先必須。
- **プローブ(R5)**: 「条件検索はマージ先が必須」(裁定③)を是正前に実測 — **不合格**
  (CanRunSearch = IsCriteriaMethod || dest でマージ先なしでも実行可)・既存 561 全緑。
- **是正**: 共有写像 **OrganizeCriteria.FromMergeTarget**(両タブ共有=転写ドリフト防止・ECO-050 教訓)
  新設+両 VM をトグル 5 種(モック condDefs 順: hash/ext/size/name/date)へ置換+CanRunSearch=
  マージ先必須∧条件 1 つ以上+XAML 2 面をトグルチップへ置換(配置は現行レイアウト内最小 —
  3 ゾーン化は次の v2 追随 ECO)。Core・エンジン・オラクル無改変。
- **テスト対応表(移行 2・退役 2・新設 5)**: 既存「条件検索」fact 2 件(画像/作業タブ)を新意味論へ移行
  (観測点=マージ先除外・一致表示・スコープは保存)。ECO-040 の整列 fact 2 件(整理トレイ条件入力の
  ピル垂直中央)は**検査対象の TextBox が消滅**したため退役(ECO-040 観点はタグ追加検索 fact が存続担保)。
  新設= プローブ(マージ先必須×両タブ)+**写像の全数 pin**(5 条件→ SearchCriteria — 「全数がどの
  検査面にもない」様式の封止)+属性別トグル E2E(5 属性)+AND 結合。
- **機械受入(4 点全緑)**: build 0/0・**Tests 564/564**(561+新設 5 −退役 2。プローブ合格転化)・
  **Oracle 109+2skip**(無改変)・validate_bom 0-0・selftest OK。
- diff: src 5(共有写像新設+両 VM+XAML 2)・tests 3(新設 1+移行 2+退役)。

- **GF-055-01(golden 所見 2026-07-06 → 是正済み 2026-07-07)**: マージ先+条件 ON でも「似た画像を
  探す」が押せず、類似タブ往復で直る。真因= ホスト転送セッター(CondX)が子 VM へ値を渡すだけで
  **ホスト自身の PropertyChanged を発火しない**(XAML はホストにバインド・タブ往復で直るのは
  SetSearchMethodCommand のホスト一括通知のため。**ECO-038「転送殻の通知漏れ」の同型を本 ECO の
  実装自身が再発させた**)。是正= 転送セッターの全通知化(CR-6 先例)+回帰プローブ(ホスト経由の
  トグル変更で PropertyChanged 発火+CanRunSearch 即時反映 — 是正前不合格を実測)。作業タブは
  VM 直バインドのため非該当。機械受入再走= Tests 565/565・Oracle 109+2skip・validate 0-0。

### golden 合格基準(gate② — maintainer 実機・GF-055-01 是正後の再確認)

1. 整理モード → マージ先を 1 枚選ぶ → 条件検索タブ → **トグル 5 種(ハッシュ値/拡張子/サイズ/
   ファイル名/更新日)**が表示される(自由入力 2 欄は無い)。
2. **ハッシュ値**トグル ON で検索 → マージ先と完全同一のファイル(コピー)だけが候補に出る。
3. **ファイル名**トグル ON → 「orientation_fixture_06」系の複製群(rot90/mirror 等)が候補に出る。
4. **マージ先未選択では「似た画像を探す」が押せない**(条件検索でも — 裁定③)。トグル全 OFF でも押せない。
5. 回帰: 類似画像検索(しきい値 70)と作業タブの整理トレイが従来どおり。
