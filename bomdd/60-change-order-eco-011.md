## 3. 設計契約改訂マニフェスト

### 表示軸モデル(確定)
コレクション = 選択スコープ(REQ-053・コレクションファースト・横断しない=不変)。その**内側**で:

| 要素 | 内容 |
|---|---|
| **FS フォルダ軸** | OS サブフォルダ階層を `relative_path` から派生して潜る。ルート=コレクション直下(直下サブフォルダ + relative_path にディレクトリ成分を持たない画像)。チップ/フォルダカードで潜り、パンくずで戻る。読み取り専用(INV-009) |
| **タグビュー軸** | 選択ビューのタグ階層(NodeGraph・AND パス意味論)を E-VIEWSVC-009 から消費。チップ+パンくずで潜る/戻る(ECO-010/I01 でツリー→チップ化済) |
| **表示軸セレクタ** | ツールバーで FS フォルダ ⇄ タグビュー を切替(既存「表示切替」を本モデルへ正式化)。軸切替で階層/フィルタをリセット |
| **「全画像」固定入口** | **維持**。選択コレクション内の全 normal 画像をフラット表示(条件なし)。2軸の構造ナビとは別の「とにかく全部」入口 |

- 縮退: `include_subfolders=false` のコレクションは FS 軸ルート=フラット(全画像と等価)。
- スコープ不変: いずれの軸も**選択コレクション内**に限定(複数コレクション横断はしない=REQ-053)。

## 6. provenance / lesson 連結
- 本 ECO は ECO-010 が UQ-I04 解決後へ繰り延べた**表示軸概念の確定**。I01(タグビュー軸ナビのチップ化)と一対で、**原典 view-prism のフォルダ+パンくず**を復権する(本実装が左ペインツリーへ替えた意図的差分の差し戻し)。
- lesson: 新 CAD は**本実装が原典から離れて入れた『意図的差分』をも差し戻す**ことがある([[mock-ui-ir-is-cad]])。差し戻しが**データモデルで既に支持されている**(relative_path 格納済)場合、新スキーマ/スキャンなしの派生ナビとして低リスクに実現できる — UI-IR 抽出で「既存データが何を支持するか」を確認することが、差し戻しの実現コストを見積もる鍵。
