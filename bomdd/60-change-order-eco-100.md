## 4. 是正方針(着手時確定)+裁定(a)〜(e)

### 4.1 是正方針(案)

1. **挿入表示の駆動を一般化**: `Editor.IsPlacing`(クリック配置)に加え `Editor.DraggingNode`(行移動)/
   ドラッグ配置中を包含する「挿入表示可視」状態を導入し、挿入ポイント・「＋ 子にする」・一時停止
   (⋯/家非表示)・ヘッダ帯を同一バインドで駆動(視覚はクリック配置と同一=発明なし)。
2. **ドロップ受け**: 挿入ポイント/「＋ 子にする」面に `DragDrop.AllowDrop`+DragOver/Drop を配線。
   ドロップ意味論はクリック配置と同一関数へ合流(兄弟=InsertBefore 位置/子=末尾+自動展開)。
   旧 `OnTreeDrop`(行上=子/空白=ルート)は撤去し挿入表示経由へ一本化。
3. **行移動 API**: `Editor.MoveNode(node, parent, index)`= サブツリーごと付け替え(EditNodeViewModel を
   そのまま移設=per-placement 状態随伴)・循環ガード(IsInSubtree)・同位置 no-op・SetDirty・移動後選択。
4. **行ドラッグ開始**: 行全体 PointerPressed→移動閾値 4px(第一便のパレット判別様式を再利用)→
   `DoDragDropAsync`(NodeIdFormat)。クリック(閾値未満)は従来どおり展開/選択。
5. **プローブ(R5)**: スタブ赤→緑転(MoveNode 意味論・循環ガード・同位置 no-op・状態随伴・
   挿入表示の出現/消滅・禁止位置の非表示)。
6. **R7**: ECO-099 の headless+Skia 撮影ハーネスを再利用し、ドラッグ中状態を P2-placing.png と並置
   (「同一」検証)+実機スクリーンショットを read-across 材料として納品。

### 4.2 裁定(a)〜(e)(本 ECO で確定・2026-07-16。依頼文の既定方向を採択)

| 項 | 裁定 | 根拠 |
| --- | --- | --- |
| (a) ドラッグ起点 | **行全体ドラッグ+移動閾値 4px**(子行の ⠿ はドラッグ可能の視覚シグナルとして mock どおり表示継続)。⠿ を親行へ出す案は採らない(視覚面の変更= mock 改版側) | 第一便でパレットカードに確立した判別様式の再利用。mock は子行 ⠿ のみ・親行は沈黙 |
| (b) 禁止ドロップ | **自分自身・自分の子孫への入れ子は禁止(循環防止)**。禁止位置には挿入ポイント/「＋ 子にする」を**出さない**(グレーでなく非表示=クリック配置の「非配置時に出ない」と同型)。同位置へのドロップ= no-op(ツリー不変・ダーティ不変) | 循環はデータ破壊。非表示様式は VC-TAG-12⑤ の視覚言語と一貫 |
| (c) キャンセル | **Esc・有効位置外ドロップ=キャンセル(ツリー不変)**。ドラッグ終了で挿入表示・状態表示が一切残らない(VC-TAG-12⑤ 同型) | DoDragDropAsync の native キャンセル+状態クリーンアップ |
| (d) 移動の意味論 | **サブツリーごと移動**・per-placement 状態(別名・条件・展開モード・HOME)は移動後も保持 | EditNodeViewModel 構造が状態をノードに随伴済み(§3.1-4)。HOME はノード id 基準=移動不変 |
| (e) ドラッグ中の状態表示 | **ヘッダ帯=配置中と同型**。パレットからのドラッグ配置=既存「「{名}」を配置中…」を再利用(意味論同一=新文言なし)。**行移動=新文言「「{名}」を移動中 — 光った位置にドロップ」(ja)を起こす**(en= "Moving \"{tagName}\" — drop on a highlighted spot")→ copy を ECO へ記録し golden で裁定を仰ぐ。ドラッグ元行は減光(既存 D&D 言語の範囲)・独自ゴーストは作らない | 帯の切替様式は VC-TAG-12② の再利用。移動中 copy は mock に無いため golden 裁定+read-across で CAD へ |
