## 4. 是正方針(案・着手時確定)

1. **次元① 重複キー lint**: ja/en の生テキストから行頭キーを抽出し重複 0 を pin
   (CpI18n010AssetTests へ追加・現状 0 なので即緑=恒久ガード)。
2. **次元② 未使用キー lint+棚卸し(本 ECO の主作業)**:
   - (a) T()/Loc[] の**非リテラル引数の全数調査**で動的構築プレフィックス registry を作る(テスト内宣言)。
   - (b) lint= 「全キーは src に完全一致で出現する か registry のプレフィックスに一致する」。
   - (c) **棚卸し= lint が照らした本物のデッドキーを ja/en から削除**(推奨。粗 652 の精査後)。
     代替= baseline 凍結(既存違反を許容し新規のみ fail)— 資産の 53% ノイズが残るため非推奨だが、
     削除規模が暴れた場合の退避先として保持。
   - 削除の安全網= REQ-050 フォールバック(誤削除は生キー表示で顕在化)+全テスト緑+R7 主要面。
3. **次元③ 解決タイミング lint**: `= _localization.T(`/`= ErrorMessages.Resolve(` の**プロパティ代入**を
   .cs 走査で検出。無害クラス(モーダル一時表示・毎回再計算ラベル)は**根拠コメント付き allowlist**
   (ECO-106 §3 の層別を初期値)。新規違反= fail(常駐メッセージの再発を機械で止める)。
4. **プローブ(R5)**: ①= 人工重複を一時挿入して赤→撤去(検出能力の実証)/②= 既知デッドキー
   (hierarchy.addRootNamed 等 6 件= ECO-099 記帳)で赤→棚卸しで緑/③= ECO-104/106 の是正前
   イディオム(git 履歴の実在例)を検出することをフィクスチャで実証。
5. **CP/M-BOM**: CP-I18N-010 へ 3 次元を宣言・M-I18N-011 受入観点へ追記(M4)。
