## 4. 是正方針(案・着手時確定)

1. **次元① 重複キー lint**: ja/en の生テキストから行頭キーを抽出し重複 0 を pin
   (CpI18n010AssetTests へ追加・現状 0 なので即緑=恒久ガード)。
2. **次元② 未使用キー lint+棚卸し(本 ECO の主作業)**:
   - (a) T()/Loc[] の**非リテラル引数の全数調査**で動的構築プレフィックス registry を作る(テスト内宣言)。
   - (b) lint= 「全キーは src に完全一致で出現する か registry のプレフィックスに一致する」。
   - (c) **棚卸し= lint が照らした本物のデッドキーを ja/en から削除**(推奨。粗 652 の精査後)。

     削除規模が暴れた場合の退避先として保持。
