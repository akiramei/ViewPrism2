# Change Order — ECO-008(タグ種別の作成後変更 — 定義値保持+付与ガード)

> タグタブ UI/UX モックレビュー(maintainer)で「**タグ種別は作成後も変更可能**」を確定。現実装は作成後ロック([TagEditorViewModel.cs:12,35](../src/ViewPrism2.App/ViewModels/TagEditorViewModel.cs#L12) 「種別(既存は変更不可)」)で、仕様 §2.6 commonAttrs「種別: 作成後も変更可能」と乖離していた。

## 3. 振る舞い契約(spec 決定 — 製造パッケージの中核)
- **R1(解錠)**: タグ種別は**作成後も変更可能**(REQ-020 実現)。新規・編集の双方で種別セグメントを操作可能にする。ただし R2 ガードに従う。
- **R2(付与ガード)**: 当該タグに**付与(`image_tags`)が1件でも存在する場合、種別変更を拒否**(`ValidationError`)。UI は種別セグメントを**非活性+理由表示**(「付与済みのため種別を変更できません。先に付与を解除してください」)。データ損失ゼロ・非破壊志向(`INV-009` 系の思想)に整合。
- **R3(定義保持)**: 種別変更で**元種別の定義設定**(textual 候補値 / numeric min/max/step/unit)を**破棄せず保持(非表示)**。同種別へ戻すと復元。
  - 保持範囲は最低でも編集ダイアログ・セッション内で復元可能とする。**永続保持(スキーマが両設定を併存保持できるか)は製造判断**(§5)。
- **R4(原子性)**: 種別変更は単一トランザクション・原子(`INV-006` 整合)。

## 8. provenance / lesson 連結
- 本 ECO は ECO-007 と同じタグタブ UI-BOM round(`bomdd/ui/`)の派生。認識合わせの過程で「1ダイアログ登録」の期待に**作成後の種別変更**が含まれることが判明し、現実装のロック(TagEditorViewModel)と仕様「変更可」の乖離を捕捉した。
