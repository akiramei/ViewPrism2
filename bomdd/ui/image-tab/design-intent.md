# 設計意図(Design Intent)— 画像タブ

> **由来**: モック(CAD)の **DESIGN DIRECTION バンド**(`ViewPrismUI:資料/画像タブ/ViewPrism2 画像タブ.dc.html` 冒頭)+ **`ViewPrismUI:docs/01_design_direction.md`**(ViewPrism2 共通 UI 方針)から抽出した **権威ある設計意図**。
> **位置づけ**: モック+UI-IR=CAD([[mock-ui-ir-is-cad]])。本書はその CAD の設計意図欄に相当し、製造(実機)が忠実に再現すべき対象。BomDD `method/ui-ir-ui-bom.md` 原則3(設計意図は nonBom でなく designIntent)に従い、テーマ/思想を **上位部品** として抽出する。
> **継承**: タグタブで確立した統一デザイン言語([../design-intent.md](../design-intent.md))を土台とする。本書は画像タブ固有の設計意図(二軸ブラウズ・潜る/戻る統一・インライン値入力・モードペイン)を追補する。

## 1. 思想(DESIGN DIRECTION)

> 「**同じ画像を、フォルダでもタグでも掘る。ひとつの操作で、ふたつの視点。**」
> コレクション(物理フォルダと1対1)を起点に、**ファイルシステム**軸でも、タグタブで組んだ**ビュー**軸でも同じ手触りでブラウズする。チップを押して潜り、パンくずで戻る。タグ編集はモードとして切り出し、値入力はポップアップを廃してインラインに。

タグタブの思想「**ひとつの光を、整理された視点へ。バラバラなUIを、ひとつの思想で束ねる。**」を画像タブが受け継ぐ。タグタブが作った「ビュー(視点)」を、画像タブが**閲覧軸として消費**する — これが「ひとつの思想で束ねる」の具体的な接続点。

## 2. 画像タブ固有の設計要求(原則として扱う)

| # | 原則 | 内容 | 含意 |
|---|---|---|---|
| I-01 | **二軸ブラウズ** | 同じ画像群を FS 軸 / タグビュー軸で見る。軸はツールバーの「表示」で切替 | 軸切替を第一級操作に。軸ごとにチップの意味(絞り込み/ナビ)が変わる |
| I-02 | **潜る/戻るの統一** | フォルダ階層もビュー階層も、**チップで潜り・パンくずで戻る**同じ操作 | パンくず・ナビチップを両軸で共通部品化(その場分岐禁止) |
| I-03 | **編集はモード** | 通常閲覧と編集を分け、編集モード時だけ選択・右パネルを有効化 | 右ペインは常時でなく文脈ペイン。閲覧時クリック=ビューア(本スコープ外)、編集時クリック=選択 |
| I-04 | **値入力はインライン** | タグ値入力はポップアップを廃し、右パネル内で行を展開して付与 | テキスト=候補値チップ列、数値=範囲セル。モーダルを増やさず文脈を保つ |
| I-05 | **クロムは静かに、画像が主役**(共通) | 中央のサムネ一覧を主役化、左右ペインと周辺は控えめ | 低彩度の周辺・触れる余白(行/カード)・密度を保つ |

> タグタブ4原則(共通の空間モデル / 階層が主役 / ひとつの部品言語 / 触れる余白)は[../design-intent.md](../design-intent.md)を参照。画像タブも全面的に従う(同じ3ペイン骨格・同じ行/カード/チップ/CTA・44px 基準)。

## 3. デザイントークン(CAD 由来 — タグタブと同一基盤)

トークンはタグタブと**完全共有**(青基調 `#2F6BED`・選択淡青・IBM Plex・44px 行・角丸スケール)。[../design-intent.md §3](../design-intent.md) と `ViewPrismUI:docs/01_design_direction.md` が正典。画像タブ固有の追補のみ記す。

| 用途 | 値 | 備考 |
|---|---|---|
| コレクション選択面 | 枠 `#CFE0FC` / 面 `#F3F8FF` / アイコン面 `#E3EEFD` | タグタブのビュー行選択と同一言語 |
| FS 軸アイコン | 面 `#EEF1F5` 文字 `#5B6473`(中立グレー) | ファイルシステム=静かな軸 |
| タグビュー軸アイコン | 面 `#FDEEDE` 文字 `#C47D18`(琥珀) | サンプルアルバム(ビュー)=タグ由来の暖色 |
| グリッド選択 | 枠 `#2F6BED` + リング `rgba(47,107,237,.22)` + チェック面 `#2F6BED` | GF-02: セル枠+チェックのみ(行背景は出さない) |
| リスト選択行 | 面 `#EAF1FE`(淡青) | GF-02: リスト行の選択視覚は正 |
| サムネタグドット | タグ9色(E-DESIGN-028)+ 白リング `0 0 0 1.5px #fff` | 閲覧時のみ・最大3 |
| 数値インライン | 琥珀 面 `#F7F9FC`/枠 `#EEF0F4`・現在値 `#9A7B1A` | タグタブ amber(数値型)と整合 |

> **タグ9色(E7 確定・共有)**: `#E5484D #F2912B #E8B931 #30A46C #12A594 #2F6BED #8B5CF6 #E93D82 #5B6473`。モックのタグ定義(サンプルアルバム橙/おすすめ紫/お気に入り赤/職種青/アーツ青緑/性別ピンク/評価黄)はこの9色から採られている。

## 4. 部品言語(原則: 全機能で使い回す)

画像タブは ECO-009 で正典化した部品言語(行カード/チップ/CTA/アイコンボタン/セグメント/カラードット/種別チップ)を**再利用**し、画像タブ固有の surface 部品を**追加**する。詳細は [design-system-bom.md](design-system-bom.md)。

| 部品 | 仕様(CAD) | 由来 |
|---|---|---|
| **コレクション行カード** | アイコン面+名前+パス muted+件数バッジ・選択=淡青枠面 | タグタブ ビュー行と同型(再利用) |
| **表示軸セレクタ** | アイコン+ラベル+▼、ドロップダウン(ラベル+sub+チェック) | **新規** |
| **パンくず** | ホーム+›+クラム(最後濃/中間 muted)+件数 | **新規**(両軸共通) |
| **タグチップ(絞り込み/ナビ)** | 色ドット+ラベル+件数 pill +(ナビ時)› | タグタブ chip 言語の variant |
| **画像グリッドカード** | 正方サムネ+名前・タグドット(閲覧)・チェック+リング(編集) | **新規**(E-UI-GRID-022 セル) |
| **画像リスト行** | ミニサムネ+名前+サイズ+更新日・選択行淡青 | **新規**(E-UI-GRID-022 行) |
| **タグ編集パネル** | ヘッダ(編集+枚数pill+×)・現在のタグ/タグ追加タブ | **新規**(E-UI-TAGASSIGN-029) |
| **現在タグピル** | 色ドット+名前+×・種別色配色 | タグタブ条件チップと同型 |
| **タグ追加行** | 色ドット+名前+(✓/キャレット/+)・種別グループ | **新規**(種別見出しは種別チップ再利用) |
| **インライン値ピッカー** | テキスト=候補値チップ列 / 数値=範囲セル | **新規**(原則 I-04) |

## 5. 既存 E-BOM との関係(設計意図レベル)
