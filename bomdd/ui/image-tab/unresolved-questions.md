| ID | 決定 | 根拠(なぜ差し戻すか) | E-BOM 影響(ECO-010 で同期) |

| UQ-I01 | **モック採用** — タグビュー軸ナビは中央の**軸セレクタ+ナビチップ+パンくず**(左ツリーを廃止) | モックの思想「ひとつの操作で、ふたつの視点」。FS 軸と view 軸を**同一の『チップで潜る・パンくずで戻る』**で扱うのが核心。view 軸だけ左ツリーにすると二軸対称が壊れ思想が死ぬ。階層編集はタグタブ中央に残るため画像タブは「ビューを消費しナビゲート」のみ。深い階層の俯瞰性はパンくず(フルパス)で担保 | **E-UI-NODEGRAPH-025**: 名称/スコープを「タグタブ中央のビュー階層エディタ」に絞る。**画像タブ左 NodeGraph ナビゲーションツリーを撤回**。**ECO-006/BL-001(画像タブ NodeGraph ナビのルート表示規律)も撤回**(対象が消滅)。画像タブのビューナビは E-UI-GRID-022 周辺へ |
| UQ-I02 | **モック採用** — テキスト候補値/数値の値入力は右パネル**インライン**(ダイアログ廃止) | 設計方針 I-04「ポップアップ廃止」。固定値・候補値選択はインラインが明確に上(文脈保持・モーダル削減)。設計意図そのもの | **E-UI-TAGASSIGN-029**: 「numeric 固定値ダイアログ」をインライン値ピッカー(候補値チップ/範囲セル)へ改訂。REQ-027 原子バッチ適用は維持 |
| UQ-I02b | **連番を別アクションで存置** — インライン固定値とは別に「連番で付与」の入口を残す | ビューアに見開き/綴じ方向(spread-right/left)があり**マンガ/連番ページ管理**が射程。連番=ページ番号付与は実需が高い。インライン範囲セルは「全選択へ同一値」しか表せず連番を表現できないため、別動線で機能を保持 | **E-UI-TAGASSIGN-029**: 連番付与を独立アクション(小ポップオーバー or インライン連番設定)として残す。REQ-027 連番計算は維持 |
| UQ-I05 | **モック採用** — グリッドセルは**サムネ+名前のみ(サイズ除去)**。サイズはリスト列/詳細で担保 | [[mock-ui-ir-is-cad]]: 原典 view-prism 準拠より新 CAD(モック)が正典。グリッドは画像が主役でサムネ下サイズは視覚ノイズ。「視覚スキャン=グリッド/詳細メタ=リスト」の情報設計は妥当(OS のサムネ表示も同様)。dedup でサイズを見たい局面はリスト/類似機能側で担保 | **E-UI-GRID-022 / DC-GRID-001**: ECO-004 で復活させたグリッドセル size を差し戻す(リスト列のサイズは維持)。「過去是正の差し戻し」根拠を ECO に明記 |
