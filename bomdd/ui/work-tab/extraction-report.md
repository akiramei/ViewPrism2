### 作業スペースの定義(DOM-0025)
名前付き・永続・ユーザー管理の画像コンテナ。属性 = `id / name / isDefault / 所属画像[]`。
- **不変条件**: 同時にデフォルトは厳密に1つ。
- **デフォルト** = 画像タブ作業モード「追加」の*自動追加先*。
- **＋(追加)** = デフォルトを回転 — 新規空デフォルトを作り、旧デフォルトを**時刻名**で「保存済みスペース」へ降格。
- デフォルト以外は**リネーム可**。スペース間で画像を**移動可**(作業モード+「別スペースへ移動」)。
- 削除画像(INV-009 ソフト削除)は件数・一覧から除外。

## 4. 画像タブとの構造差(4点)

1. 左サイドバー = 作業スペース(コレクションでない)。
2. **表示軸セレクタ(FS/サンプルアルバム)を持たない** — フラットなスペース表示。DCLogic に軸/パンくずの残骸はあるが template 未描画・context() 無視。
3. パンくずの代わりに**ワークスペースヘッダ**(アイコン+名前+既定（自動追加先）+件数)。
4. 作業モードの随伴ボタンが「追加」でなく**「別スペースへ移動」**(現スペース→他スペースへ membership 移動)。
