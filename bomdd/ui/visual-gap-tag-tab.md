# 視覚ギャップ分析 — タグタブ(製造品 vs CAD)

> **位置づけ**: モック(`work/tag-tab/`)+ UI-IR(`bomdd/ui/`)= **CAD(権威ある設計原器)**。本書は **稼働中実機(製造品)を CAD と視覚で突合した製造検査**である。ECO-007(E1〜E7)は表示契約のフィールド差のみを是正したが、**設計言語(カード/チップ/CTA/余白/欠落表示)の乖離**を取りこぼしていた。本書がその全量を棚卸しする。
> 生成: 2026-06-16 / 参照: 実機=`src/ViewPrism2.App/Views/TagsTabView.axaml`・`TagEditorWindow.axaml` + 実機スクショ / CAD=M1(タグ管理)・M2(作成ダイアログ)・M3(配置タグ設定)+ それぞれのスクショ。

## 0. 診断サマリ

実機は**機能を満たす最小スケルトン**(素の Grid+DockPanel・グリフ文字ボタン・条件は muted テキスト・候補値非表示)。CAD は**カード基調の設計言語**(枠付きカード・色付きチップ・条件チップ・CTA・件数/凡例・ヘルプ箱)。差は **Avalonia の限界ではなく、デザインシステム(コンポーネント層)が未構築**であること。

重大度: **S1**=欠落/構造(表示要素そのものが無い・コンテナ構造が違う) / **S2**=設計言語(カード化・色チップ・CTA・アイコン・余白) / **S3**=細部。

## 6. 根本=未構築のデザインシステム部品(E-DESIGN-028 / K-DESIGN)

ほぼ全 gap は以下の**共通部品が未定義**であることに帰着する。これらを E-DESIGN-028 に定義し各 surface へ適用するのが本丸。

| 部品 | 用途 | 現状 |
|---|---|---|
| **Card / Surface**(枠+角丸+余白+影) | ビュー行・パレット行・階層コンテナ・ダイアログ各パネル | 無(素 Border/DockPanel) |
| **CTA ボタン**(青グラデ・44px・影) | 新規ビュー・追加・タグを配置 | 無(既定 Button) |
| **型チップ**(色分け: シンプル灰/テキスト青/数値緑) | パレット・階層行・ダイアログ | 一部(枠 muted のみ・配色なし) |
| **条件チップ**(型別配色・じょうご・regex mono) | 階層の条件表示 | 無(muted テキスト) |
| **候補値チップ / 範囲サマリ** | パレットカード | 無 |
| **バッジ/件数 pill** | タグ数・件数・"Nタグ" | 一部(タグ数バッジのみ・E1 で追加) |
| **アイコンボタン**(ベクタ+hover) | 行内 編集/条件/削除/ホーム/ドラッグ | 無(グリフ文字) |
| **節見出し/凡例/ヘルプ箱/ドラッグハンドル** | 各ペイン | 概ね無 |

## 7. 重大度ロールアップと推奨スコープ

- **S1(欠落/構造)**: 階層コンテナカード・条件チップ化・パレット候補値/範囲表示・ダイアログ2カラム・親行の型チップ — **機能的にも見えていない情報がある**(条件・候補値)。最優先。
- **S2(設計言語)**: カード化・CTA・色チップ・アイコンボタン・凡例 — 量が多い本体。
- **S3(細部)**: 節見出し・「=元名」・HOME pill・ヘルプ箱。

**推奨**: 本書を入力に **ECO-009「タグタブ デザイン言語移植(原典素朴 → CAD)」** を起票。中核は §6 のデザインシステム部品を E-DESIGN-028/K-DESIGN に定義し、3ペイン+ダイアログへ適用。golden 基準を**モックに切替**し、**golden-in-the-loop**(反復ごとに実機スクショを M1/M2/M3 と突合)で製造する。E1〜E7 は刷新後も整合(★撤去・タグ数・9色・プレビュー核ヘルパ・条件は元々チップ化対象)。

> 注: 製造順は S1(欠落=条件チップ・候補値表示・コンテナ)→ デザインシステム部品(§6)→ 各 surface 適用、が安全(情報欠落を先に塞ぎ、その後に外装)。
