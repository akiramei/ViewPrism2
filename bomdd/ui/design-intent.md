# 設計意図(Design Intent)— タグタブ / Ver2 デザイン言語

> **由来**: モック(CAD)の **DESIGN DIRECTION 01/02/03**(`work/tag-tab/` M1/M2/M3 冒頭)から抽出した **権威ある設計意図**。
> **訂正履歴**: 初回抽出(2026-06-16)で本バンドを `ui-ir.json` の unmodeledElements に「プレゼン用マーケ・nonBom」と誤分類し破棄していた。maintainer 指摘により**設計意図そのもの**として再抽出(本書)。これは「ひとつの思想で束ねる」=統一デザイン言語の憲章であり、K-DESIGN / E-DESIGN-028 / ECO-009 の基盤。
> **位置づけ**: モック+UI-IR=CAD([[mock-ui-ir-is-cad]])。本書はその CAD の設計意図欄に相当し、製造(実機)が忠実に再現すべき対象。

## 1. 思想(DESIGN DIRECTION 01)

> 「**ひとつの光を、整理された視点へ。バラバラなUIを、ひとつの思想で束ねる。**」
> 機能ごとに個別設計されていた画面を、**「ナビ → 一覧 → 詳細」という共通の3ペイン**と、**共通のヘッダー・行・カード・チップ**で統一する。どのタブに移っても**操作の手触りが変わらない** — それが Ver2 の土台。

- DD02: 「**3種別を、ひとつのダイアログに**」(シンプル/テキスト/数値を適応型ダイアログへ統合)
- DD03: 「**置いたタグに、文脈を与える**」(配置に別名・条件・ホームを重ねる)

## 2. 4原則(設計要求として扱う)

| # | 原則 | 内容 | 含意 |
|---|---|---|---|
| 01 | **共通の空間モデル** | 全タブが同じ3ペイン。位置で機能を覚えられる | ナビ→一覧→詳細を全タブ共通に |
| 02 | **階層が主役** | タグツリーを中心に、クロムは静かに脇へ | 中央を主役化・周辺は控えめ(低彩度・小コントラスト) |
| 03 | **ひとつの部品言語** | 同じ行・カード・チップを全機能で使い回す | **再利用コンポーネント層を正典化**(その場スタイル禁止) |
| 04 | **触れる余白** | 行は44px。タブレットでも迷わず押せる | タッチ前提の余白・44px 行・十分なパディング |

## 3. デザイントークン(CAD 由来)

### 3.1 カラー(青基調・清涼)
| トークン | 値 | 用途 |
|---|---|---|
| primary | `#2F6BED` | アクセント主・選択・リンク・CTA 基色 |
| primary-on-pale | `#2459CF` | 淡背景上の青文字(テキスト型チップ等) |
| primary-light | `#5B8DEF` | グラデ上端・補助 |
| pale-blue(選択面) | `#EAF1FE` / 選択背景 `#F3F8FF`・`#F6FAFF` / 選択境界 `#CFE0FC` | 選択・淡い青の面 |
| CTA グラデ | `#4A82F0 → #2F6BED`(hover `#3F76EB → #255ED6`) | 主要ボタン |
| page-bg | `#EAEDF1` / `#EEF1F4` | 画面背景(やや青み) |
| surface | `#FFFFFF`、境界 `#E6E9EF`・`#E8EBF0`、hover 境界 `#CFD6E1` | カード/パネル |
| text | 主 `#1C2230`・`#2B3340` / 弱 `#566072`・`#6B7480` / 微 `#98A0AD`・`#AAB1BD` | 文字階調(非アクティブ=微弱グレー) |
| amber(数値/警告) | 面 `#FBF3DF` 文字 `#9A7B1A` / warn `#B5670C`・`#D97A18` | 範囲・ホーム・要確認 |
| green(数値型) | 面 `#EAFAF3` 文字 `#0F8A5E` | 数値型チップ・条件(候補値) |
| gray(シンプル型) | 面 `#F0F2F6` 文字 `#5B6473` | シンプル型チップ |
| タグ9色(E7) | `#E5484D #F2912B #E8B931 #30A46C #12A594 #2F6BED #8B5CF6 #E93D82 #5B6473` | タグ識別色プリセット |

> **状態色の意図(maintainer 指摘の核心)**: **選択=淡い青、ホバー=淡い境界、非アクティブ=淡いグレー**。状態が淡い面で一目で分かる。
> **実機との差**: 現行アクセントは `#3B82F6`(Tailwind blue-500)で **モック `#2F6BED` と別の青**。要統一。

### 3.2 タイポグラフィ
- **フォント: IBM Plex Sans / IBM Plex Sans JP / IBM Plex Mono**(「**Latin と日本語を同一設計で統一**」)。mono は regex 等。
- **実機との差(重大)**: 実機は **Inter**(K-DESIGN・csproj `Avalonia.Fonts.Inter`)。**モックの IBM Plex と不一致** → ECO-009 で IBM Plex へ移行。
- サイズ目安: 見出し 30px/700(ページ)・15-16px/600(セクション)・本文 14px・キャプション 11-12px・コード(mono)13.5px。letter-spacing 見出し -.01em。

### 3.3 余白・寸法・角丸・陰影
- 余白: 4px グリッド(原則04)。行 **44px**(リーフ 42 / ビュー行 46)。パディング カード 11-18・行 8-14。
- 角丸スケール: チップ 5-8 / カード 11-14 / ボタン 10-11 / バッジ 6-9 / 入力 10。
- 陰影: カードは極淡(`0 1 2 rgba(20,32,64,.04)`)、CTA は色付き淡影、選択カードに薄い青影。

## 4. 部品言語(原則03 — 全機能で使い回す)

| 部品 | 仕様(CAD) |
|---|---|
| **ヘッダー** | セクション見出し 15-16px/600 + 補助 muted。ペインヘッダ高 58-62px。 |
| **行(row)** | 高44px・色ドット+名前+型チップ+(右)アクション。選択=淡青面。 |
| **カード(card)** | 白面+境界 `#E8EBF0`+角丸12-14+淡影。選択=淡青面+青境界+薄影。ホバー=境界 `#CFD6E1`。 |
| **チップ(chip)** | 型チップ(シンプル灰/テキスト青/数値緑)・条件チップ(型別配色+じょうご・regex は mono)・候補値チップ(灰枠)・範囲ピル(amber)。角丸6-8。 |
| **CTA ボタン** | 青グラデ・白文字・44px・角丸10-11・色付き淡影。 |
| **アイコンボタン** | ベクタ(PathIcon)・28px・hover で淡面。グリフ文字直書きは不可。 |
| **バッジ/件数 pill** | 灰面・角丸6-9。 |
| **ナビ** | セグメント pill(アイコン+ラベル・active は白面+影+青文字)。 |

## 5. 実機との主要乖離(再抽出で顕在化)

| 観点 | CAD(意図) | 実機(現状) | 是正 |
|---|---|---|---|
| 思想 | ひとつの部品言語で統一 | Fluent 既定+その場スタイル | 再利用コンポーネント層(済: パレット実証)を全体へ |
| アクセント青 | `#2F6BED` | `#3B82F6` | 統一 |
| フォント | **IBM Plex** | **Inter** | IBM Plex へ移行 |
| テーマ適用 | 全コントロールに意図継承 | コントロール既定のまま | **FluentTheme グローバル上書き**(TextBox/Button/ComboBox/ListBox 選択/角丸) |
| 状態色 | 選択=淡青/非アクティブ=淡灰 | Fluent 既定の選択色 | トークン化して全体適用 |
