# ECO-119: CAD 記述誤り — image_tab.md 行 107 が存在しない「タグ編集の docked 付与操作」を不変条件の実例に挙げている

- 起票日: 2026-07-20
- 報告者: ECO-116 §6-1(診断中のスコープ外所見)からの分離起票
- baseline: main `9f4f972`(ViewPrismUI: `2c31176`)
- 種別: CAD 記述誤りの是正(doc-only 見込み・実装/視覚は不変)

## §1 症状

CAD `../ViewPrismUI/docs/screens/image_tab.md` 行 107(レイアウト不変条件):

> **右ペイン(文脈モード): ヘッダ固定/本体スクロール/docked アクション常時可視**:
> **タグ編集の付与操作**や整理トレイの「マージを実行」は下部 docked で常に押せ、
> タグ候補・整理対象が何件増えても押し出されない

**タグ編集パネルに docked 付与アクションは存在しない**(原器にも実装にもない)。
不変条件そのもの(docked アクションを持つ面では常時可視)は健全だが、**実例が事実に反する**。

発見経緯: ECO-116(タグ編集パネルのスクロール到達性)の工程診断で
「パネル下部に高さを食う docked 要素があるのでは」を潰す際に突き合わせて発覚。
同 ECO では R3 としてスコープ外記録に留めた(§6-1)。

## §3 切り分け済みの事実

### 3.1 確定(起票時・原器と CAD の実測)

1. **原器(mock)に docked footer が無い**:
   `../ViewPrismUI/資料/画像タブ/ViewPrism2 画像タブ.dc.html` の `panelActive` ブロック
   (行 450-546)は「タブ行 → 区切り → 現在のタグ/タグ追加の本体」で閉じ、
   下部固定領域を持たない(次のセクションは TRASH OVERLAY)。
2. **同 CAD 内の正典が「即時付与」**: 行 737「シンプル= 行クリックで即時付与」/
   行 789-791「テキスト・数値= 行を展開して候補値・値セルをクリック」。
   docked ボタンを押す操作は正典に存在しない。
3. **面間対称の work_tab.md は正しい**(= 正解が隣の面に既在・ECO-115 型):
   同ファイル行 52 が image_tab.md の不変条件を**全面継承**した上で、行 56 は
   「docked アクション(**作業=「別スペースへ移動」/整理=「マージを実行」**)は常時可視」と
   列挙し、**タグ編集を docked の例に含めていない**。
   → 逸脱側は image_tab.md 行 107 のみ。

**混入**: `e1bc4e3`(ViewPrismUI・2026-07-02)
「表示列モデル(ビュー編集+ファイル一覧)を CAD 化 + **layoutInvariant 横展開**」。
他面(view_edit モーダル/file_list)の不変条件を右ペインへ一般化して書き下ろした際、
**実例を原器で確認せずに「タグ編集の付与操作」と書いた**(横展開時の作話)。
正典側の「即時付与」は `131ce41`(2026-06-18・初版アーカイブ)から一貫しており、
**矛盾は横展開の側が持ち込んだ**。

**潜伏**: 2026-07-02 → 2026-07-20 の 18 日。
**マスキング**: 実装は同行から「ヘッダ固定/本体スクロール」だけを読み取って充足しており、
docked の不在が golden で問題化しなかった(不変条件の散文は**全体が真である前提**で
読まれるため、内部の実例だけが偽でも検出されない)。

## §4 是正方針(案・着手時確定)

**案 A(推奨・doc-only)**: 行 107 の実例を事実へ合わせる。

- docked の実例を **整理トレイの「マージを実行」(+作業モードの「別スペースへ移動」)** に限定。
- **タグ編集パネルには docked アクションが無い**(付与は行/値クリックで即時)ことを明記し、
  同パネルに適用される不変条件は「ヘッダ固定/本体スクロール(末尾到達性= ECO-116)」であると
  書き分ける。
- work_tab.md 行 56 と**対称化**(逸脱側を正解側へ寄せる)。
- 併せて §3.2 の遡及掃射を実施し、同型の作話があれば同 ECO 内で是正。

**案 B(却下候補・記録のため明示)**: タグ編集パネルに docked 付与アクションを新設して
記述に実装を合わせる。→ **原器に存在せず**、正典「即時付与」とも矛盾するため採らない。
もし docked 化を望む場合は**設計変更として別 ECO**(mock 改版が先)。
